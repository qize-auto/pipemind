# PipeMind → Hermes+ 进化路线图

## 差距全景

| 能力 | Hermes | PipeMind 现状 | 差距 |
|------|--------|-------------|------|
| 会话管理 | SQLite 持久化 + session_search | 每次启动新会话 | 🔴 |
| 多 Provider | deepseek/openai/anthropic + fallback | 只配了 deepseek | 🔴 |
| Cron 调度 | 内置 cron 引擎 | 无 | 🔴 |
| 子代理 | delegate_task | 无 | 🔴 |
| 上下文压缩 | 自动压缩中间轮次 | 手动截断 | 🔴 |
| 技能体系 | 36 技能 + Hermes Agent | 4 技能 + 本地加载 | 🟡 |
| 工具数量 | 30+ | 21 | 🟡 |
| MCP 客户端 | 原生支持 | 无 | 🟡 |
| 富终端 | TUI + 颜色 + 代码块 | 纯文本 + ANSI | 🟡 |
| 记忆系统 | MemPalace 向量搜索 | 文件 + 关键词 | 🟡 |
| 浏览器 | 内置浏览器自动化 | 无 | 🟡 |
| 多平台 | CLI/Discord/Telegram | 仅 CLI | 🔵 |

🔴 = 核心差距（缺了跑不动） 🟡 = 重要差距（缺了不好用） 🔵 = 锦上添花

## PipeMind 天生优势

有些地方 PipeMind 可以比 Hermes 做得更好，因为是 Windows 原生：

| 能力 | Hermes (WSL) | PipeMind (Windows) |
|------|-------------|-------------------|
| 启动速度 | ~7 秒 | <1 秒 |
| Windows API | 通过 /mnt/c/ 间接访问 | 直接调用 win32api |
| 系统通知 | 无 | Windows 通知 ✅ 已有 |
| 剪贴板 | 无 | ✅ 已有 |
| 进程管理 | wsl 内 | 全系统 |
| 注册表 | 无 | 可以直接读 |
| 开机自启 | 需额外配置 | Windows 计划任务 |

## 三步走战略

### 第一步：核心架构（1 周）
让 PipeMind 真正能跑起来、记得住、不崩溃。

```
优先级:
  1. 会话持久化 (SQLite)    ← 最核心
  2. 多 Provider + 自动降级  ← API 挂了能活
  3. 上下文压缩              ← 长对话不崩
```

### 第二步：能力扩展（2 周）
追上 Hermes 的主要功能。

```
优先级:
  4. Cron 定时任务          ← 后台能做事
  5. 技能扩充至 15+         ← 更多场景覆盖
  6. 子代理分工              ← 复杂任务拆解
  7. 工具扩充至 30+         ← 更多能力
```

### 第三步：超越（持续）
利用 Windows 原生优势，做 Hermes 做不到的事。

```
  8. Windows 深度集成（注册表/服务/计划任务）
  9. GUI 模式（可视化界面）
  10. 本地小模型（ollama 集成，断网也能跑）
```

## 建议

第一步做完，PipeMind 就能正式用了。第二步做完，和 Hermes 平级。
第三步才是你提到的"更强更优秀"——那是 PipeMind 独有的路，Hermes 走不了。

你觉得这个节奏可以吗？先从第一步的第 1 项（会话持久化）开始？
